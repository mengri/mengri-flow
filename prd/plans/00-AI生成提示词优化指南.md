## 项目级强制规范

### 1.1 架构约束（不可违反）

#### 分层依赖规则
```
Domain <- App <- Infra/Ports
  ↑        ↑        ↑
  │        │        │
  └────────┴────────┘
        只能单向依赖
```

**红线（严禁违反）：**
- ❌ `internal/domain` 不能 import 任何 `internal/infra` 包
- ❌ `internal/domain` 不能 import `github.com/gin-gonic/gin`
- ❌ `internal/domain` 不能 import `gorm.io/gorm`
- ❌ `internal/app` 不能 import `internal/ports/http/handler`（应反向）

**正确做法：**
- ✅ `internal/infra/persistence/mysql` 可以 import `internal/domain/repository`
- ✅ `internal/ports/http/handler` 可以 import `internal/app/service`
- ✅ 所有层都可以 import `internal/domain/entity` 和 `internal/domain/errors`

#### 依赖注入规范
- 必须使用 `pkg/autowire` 框架进行依赖注入
- 在 `init()` 函数中注册实现，不要使用全局变量初始化业务逻辑
- 接口定义在消费方（Domain层），实现方（Infra层）实现接口
- 使用编译时检查：`var _ InterfaceName = (*Implementation)(nil)`

### 1.2 错误处理黄金法则

#### 错误包装标准
```go
// ✅ 正确 - 包含上下文信息
return fmt.Errorf("userService.CreateUser: %w", err)

// ❌ 错误 - 丢失了原始错误
return errors.New("failed to create user")

// ❌ 错误 - 没有模块信息
return fmt.Errorf("%w", err)
```

**格式要求：**
```
"packageName.StructName.MethodName: action: %w"
```

#### 领域错误定义
- 在 `internal/domain/errors/` 定义哨兵错误（sentinel errors）
- 仅使用标准库，不依赖外部包
- HTTP Handler 必须映射到合适的 HTTP 状态码

```go
// 在 domain/errors/ 中定义
var ErrUserNotFound = errors.New("user not found")

// 在 handler 中映射
func handleDomainError(err error) int {
    switch {
    case errors.Is(err, domainerrors.ErrUserNotFound):
        return http.StatusNotFound
    case errors.Is(err, domainerrors.ErrInvalidInput):
        return http.StatusBadRequest
    default:
        return http.StatusInternalServerError
    }
}
```

### 1.3 日志规范

#### 日志格式要求
- 使用 `slog` 结构化日志
- 必须包含关键字段：`module`, `action`, `success`
- 错误日志必须包含 `error` 字段和 `stack`（如适用）

```go
// ✅ 正确
logger.Info("create resource",
    "module", "resourceService",
    "action", "create",
    "resourceId", resource.ID,
    "workspaceId", workspaceID,
    "success", true,
)

logger.Error("failed to create resource",
    "module", "resourceService",
    "action", "create",
    "error", err,
    "workspaceId", workspaceID,
)

// ❌ 错误 - 缺少关键字段
fmt.Printf("Resource created: %s\n", resource.ID)
```

### 1.4 性能约束

#### 通用性能要求
- API响应时间 **P99 < 500ms**（数据库查询优化）
- 支持 **100 QPS** 的并发量
- 内存使用：单个请求 **< 10MB**
- Goroutine泄漏：必须能在 `go test -run TestXXX -count=100` 下稳定运行

#### 数据库查询优化
- 禁止使用 `Select("*")`，必须明确指定字段
- 复杂查询必须添加索引（在entity层用 `gorm:"index"` 标注）
- N+1查询必须使用 Preload 或 Joins 优化
- 批量操作使用 `CreateInBatches`，批次大小 1000

#### 并发控制
- 在循环中启动goroutine必须使用信号量限制并发数
- 推荐 `golang.org/x/sync/semaphore` 或 `worker pool` 模式

```go
// ✅ 正确 - 限制并发
sem := semaphore.NewWeighted(10)
for _, item := range items {
    if err := sem.Acquire(ctx, 1); err != nil {
        return err
    }
    go func() {
        defer sem.Release(1)
        // 处理逻辑
    }()
}

// ❌ 错误 - 可能创建过多goroutine
for _, item := range items {
    go process(item) // 危险！
}
```

### 1.5 接口契约

#### HTTP API 统一格式
**请求：**
```
POST /api/v1/resources
Content-Type: application/json

{
  "name": "string(2-50)",
  "type": "enum[http,grpc,mysql]",
  "config": "object"
}
```

**成功响应：**
```json
{
  "code": 0,
  "data": {
    "id": "string(uuid)",
    "name": "string",
    "createdAt": "string(ISO8601)"
  },
  "msg": ""
}
```

**错误响应：**
```json
{
  "code": 40001,
  "data": null,
  "msg": "参数验证失败: name不能为空"
}
```

#### 字段约束
- `name`: 2-50字符，正则 `^[a-zA-Z0-9_-]+$`
- `description`: 最大500字符
- `workspaceId`: UUID格式（36字符）
- ID字段：统一使用字符串UUID
- 时间字段：ISO8601格式 `2006-01-02T15:04:05Z07:00`

### 1.6 代码复杂度限制

| 指标 | 可接受 | 必须重构 |
|------|--------|----------|
| Cyclomatic 复杂度 | ≤10 | >15 |
| Cognitive 复杂度 | ≤30 | >50 |
| 函数行数 | ≤60 | - |
| 函数语句数 | ≤40 | - |
| 行长度 | 120字符（软限制） | - |
| 生产文件行数 | ≤200（软） ≤1000（硬） | >1000 |
| 测试文件行数 | ≤300（软） ≤1500（硬） | >1500 |

**重构信号：**
- 函数超过60行 → 拆分成多个小函数
- 嵌套深度 > 3层 → 提取函数或使用早期返回
- 参数数量 > 3个 → 使用 Options Pattern

## Go后端开发规范

### 2.1 项目结构规范

#### 文件命名
- 实现文件：`user_service.go`（snake_case）
- 接口文件：`user_service_iface.go`（必须有 `_iface` 后缀）
- 测试文件：`user_service_test.go`（与被测文件同名）
- 接口定义：`internal/domain/repository/user_repository.go`（非 `_iface`）

#### 包命名
- 使用小写字母，单数形式
- 避免通用名称：`util`, `common`, `helper`
- 使用描述性名称：`user`, `resource`, `trigger`

#### 导入分组
必须分3组，用空行分隔：
```go
import (
    // 1. 标准库
    "context"
    "fmt"

    // 2. 第三方库
    "github.com/gin-gonic/gin"
    "gorm.io/gorm"

    // 3. 内部包（按模块路径排序）
    "mengri-flow/internal/domain/entity"
    "mengri-flow/internal/domain/errors"
    "mengri-flow/pkg/logger"
)
```

### 2.2 数据库操作规范

#### GORM 使用规范
```go
// ✅ 正确
func (r *userRepository) FindByID(ctx context.Context, id string) (*entity.User, error) {
    var user entity.User
    err := r.db.WithContext(ctx).
        Select("id, name, email, created_at, updated_at").
        Where("id = ? AND deleted_at IS NULL", id).
        First(&user).Error
    
    if err != nil {
        if errors.Is(err, gorm.ErrRecordNotFound) {
            return nil, domainerrors.ErrUserNotFound
        }
        return nil, fmt.Errorf("userRepository.FindByID: %w", err)
    }
    
    return &user, nil
}

// ❌ 错误
func (r *userRepository) FindByID(id string) (*entity.User, error) {
    var user entity.User
    err := r.db.Where("id = ?", id).First(&user).Error // 缺少Select、缺少deleted_at检查
    return &user, err // 错误未包装
}
```

#### 事务管理
```go
func (r *userRepository) CreateWithProfile(ctx context.Context, user *entity.User, profile *entity.Profile) error {
    return r.db.WithContext(ctx).Transaction(func(tx *gorm.DB) error {
        if err := tx.Create(user).Error; err != nil {
            return fmt.Errorf("create user: %w", err)
        }
        
        profile.UserID = user.ID
        if err := tx.Create(profile).Error; err != nil {
            return fmt.Errorf("create profile: %w", err)
        }
        
        return nil
    })
}
```

### 2.3 HTTP Handler 规范

#### Handler 结构
```go
type UserHandler struct {
    userService userservice.UserService // 接口，非实现
}

func NewUserHandler(userService userservice.UserService) *UserHandler {
    return &UserHandler{userService: userService}
}

// 路由注册在 router/ 包中，不在 handler 中
```

#### 请求处理模板
```go
func (h *UserHandler) CreateUser(c *gin.Context) {
    // 1. 日志开始
    logger.Info("handling create user request",
        "module", "userHandler",
        "action", "CreateUser",
    )
    
    // 2. 参数绑定
    var req dto.CreateUserRequest
    if err := c.ShouldBindJSON(&req); err != nil {
        logger.Warn("invalid request",
            "module", "userHandler",
            "error", err,
        )
        response.BadRequest(c, "invalid request format")
        return
    }
    
    // 3. 调用服务层
    user, err := h.userService.CreateUser(c.Request.Context(), &req)
    if err != nil {
        logger.Error("failed to create user",
            "module", "userHandler",
            "error", err,
        )
        response.Error(c, handleDomainError(err), "create user failed")
        return
    }
    
    // 4. 成功响应
    logger.Info("user created successfully",
        "module", "userHandler",
        "userId", user.ID,
        "success", true,
    )
    response.Success(c, user)
}
```

## 前端开发规范

### 3.1 Vue 3 + TypeScript 规范

#### Composition API 优先
```vue
<!-- ✅ 正确 -->
<script setup lang="ts">
import { ref, computed } from 'vue'

const count = ref(0)
const doubleCount = computed(() => count.value * 2)

function increment() {
  count.value++
}
</script>

<!-- ❌ 错误 - 避免使用 Options API -->
<script lang="ts">
export default {
  data() {
    return { count: 0 }
  }
}
</script>
```

#### TypeScript 严格模式
- 禁止 `any` 类型，使用 `unknown` 或泛型
- 接口命名使用 `PascalCase`，以 `I` 开头或描述性名称
- 使用 `type` 定义联合类型和工具类型

```typescript
// ✅ 正确
interface User {
  id: string
  name: string
  email: string
}

type UserStatus = 'active' | 'inactive'

// ❌ 错误
const user: any = {} // 禁止使用 any
```

### 3.2 Pinia Store 规范

#### Setup Store 语法
```typescript
// ✅ 正确
export const useUserStore = defineStore('user', () => {
  const users = ref<User[]>([])
  const loading = ref(false)
  
  const userCount = computed(() => users.value.length)
  
  async function fetchUsers() {
    loading.value = true
    try {
      users.value = await api.get('/users')
    } finally {
      loading.value = false
    }
  }
  
  return { users, loading, userCount, fetchUsers }
})

// ❌ 错误 - 避免 Options Store
export const useUserStore = defineStore('user', {
  state: () => ({ users: [] })
})
```

#### Store 命名规则
- 文件名：`user.ts`, `resource.ts`
- Store ID：使用复数形式（`users`, `resources`）
- 动作函数：使用动词开头（`createUser`, `updateResource`）

### 4.3 配置验证

#### 验证逻辑
```go
func (p *HTTPPlugin) Connect(ctx context.Context, config map[string]interface{}) error {
    // 1. 验证必填字段
    baseURL, ok := config["baseURL"].(string)
    if !ok {
        return fmt.Errorf("missing or invalid baseURL: %w", plugin.ErrInvalidInput)
    }
    
    // 2. 验证URL格式
    if _, err := url.Parse(baseURL); err != nil {
        return fmt.Errorf("invalid baseURL: %w", err)
    }
    
    // 3. 验证认证配置
    if auth, ok := config["auth"].(map[string]interface{}); ok {
        authType, _ := auth["type"].(string)
        if authType == "apiKey" {
            if _, ok := auth["apiKey"].(string); !ok {
                return fmt.Errorf("missing apiKey when auth.type is apiKey: %w", plugin.ErrInvalidInput)
            }
        }
    }
    
    // 4. 测试连接
    if err := p.testConnection(ctx, baseURL); err != nil {
        return fmt.Errorf("connection test failed: %w", err)
    }
    
    return nil
}

func (p *HTTPPlugin) testConnection(ctx context.Context, baseURL string) error {
    req, err := http.NewRequestWithContext(ctx, "GET", baseURL, nil)
    if err != nil {
        return err
    }
    
    client := &http.Client{Timeout: 5 * time.Second}
    resp, err := client.Do(req)
    if err != nil {
        return plugin.ErrConnectionFailed
    }
    defer resp.Body.Close()
    
    return nil
}
```

## 测试规范

### 5.1 单元测试

#### 测试文件结构
```go
// user_service_test.go
package service

import (
    "context"
    "testing"
    
    "mengri-flow/internal/domain/entity"
    "mengri-flow/internal/domain/errors"
    "mengri-flow/internal/domain/repository/mock"
)

func TestUserService_CreateUser(t *testing.T) {
    t.Parallel() // 并行测试
    
    tests := []struct {
        name        string
        input       *dto.CreateUserRequest
        mockSetup   func(*mock.UserRepository)
        wantErr     bool
        wantErrCode error
    }{
        {
            name: "success",
            input: &dto.CreateUserRequest{
                Name:  "John Doe",
                Email: "john@example.com",
            },
            mockSetup: func(repo *mock.UserRepository) {
                repo.On("Create", mock.Anything, mock.AnythingOfType("*entity.User")).
                    Return(nil)
            },
            wantErr: false,
        },
        {
            name: "duplicate email",
            input: &dto.CreateUserRequest{
                Name:  "John Doe",
```

```go
// ❌ 反模式1：在 Handler 中直接操作数据库
func (h *Handler) Create(c *gin.Context) {
    db.Create(&user) // 错误！应该调用 Service
}

// ✅ 正确
func (h *Handler) Create(c *gin.Context) {
    user, err := h.userService.Create(c.Request.Context(), req)
}

// ❌ 反模式2：忽略错误
rows, _ := db.Query(...) // 错误！

defer rows.Close() // 如果 Query 失败，rows 是 nil

// ✅ 正确
rows, err := db.Query(...)
if err != nil {
    return err
}
defer rows.Close()

// ❌ 反模式3：在循环中创建无限制 goroutine
for _, item := range items {
    go process(item) // 危险！
}

// ✅ 正确
sem := semaphore.NewWeighted(10)
for _, item := range items {
    if err := sem.Acquire(ctx, 1); err != nil {
        return err
    }
    go func() {
        defer sem.Release(1)
        process(item)
    }()
}

// ❌ 反模式4：使用全局变量存储业务状态
var globalUserCache = make(map[string]*User) // 错误！

// ✅ 正确
type Service struct {
    userCache map[string]*User
    mu        sync.RWMutex
}

// ❌ 反模式5：在 Domain 层返回 HTTP 状态码
type User struct {
    HTTPStatus int // 错误！Domain 不应知道 HTTP
}

// ✅ 正确
type User struct {
    ID   string
    Name string
}
```
