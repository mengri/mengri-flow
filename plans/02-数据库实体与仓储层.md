# 任务 02: 数据库实体与仓储层

## ⚠️ 项目规范提醒
在开始本任务前，请仔细阅读并遵守以下规范：

### 架构约束
- **严禁**跨层依赖：Domain → App → Infra（只能单向）
- **严禁**在 `internal/domain` 中引入 GORM 依赖（只在 infra 层使用）
- **必须**在 Domain 层定义仓储接口（接口由消费方定义）
- **必须**在 Infra 层实现仓储接口

### 实体设计原则
- **必须**将业务逻辑封装在实体方法中（充血模型）
- **严禁**创建贫血模型（只有 getter/setter）
- **必须**使用值对象封装复杂概念（如 Email、Password）

### 数据库操作规范
- **必须**使用 `WithContext(ctx)` 传递上下文
- **必须**使用 `Select()` 明确指定字段（禁止 `Select("*")`）
- **必须**验证外键关系和级联删除行为
- **必须**为查询字段添加索引（在 entity 中用 `gorm:"index"` 标注）

### 错误处理
- **必须**包装仓储错误：`fmt.Errorf("repoName.Method: %w", err)`
- **必须**返回领域错误（如 `domainerrors.ErrNotFound`）
- **严禁**直接返回 GORM 错误给上层

// ✅ 正确
func (r *userRepo) FindByID(ctx context.Context, id string) (*entity.User, error) {
    var user entity.User
    err := r.db.WithContext(ctx).
        Select("id, name, email, created_at").
        Where("id = ? AND deleted_at IS NULL", id).
        First(&user).Error
    
    if err != nil {
        if errors.Is(err, gorm.ErrRecordNotFound) {
            return nil, domainerrors.ErrUserNotFound
        }
        return nil, fmt.Errorf("userRepo.FindByID: %w", err)
    }
    
    return &user, nil
}
