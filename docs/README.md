# Mengri Flow 文档站点

这是一个简洁实用的文档浏览站点，适用于 `mengri-flow` 项目的文档管理。采用左右布局设计，左侧为文件导航，右侧为文档内容。

## 🚀 快速开始

### 步骤 1：启动文档站点
1. 确保已安装 **Live Server** VS Code 插件
2. 在 VS Code 中打开 `docs/index.html` 文件
3. 右键点击文件 → **"Open with Live Server"**
4. 或者点击右下角的 **"Go Live"** 按钮

### 步骤 2：访问文档站点
- 默认地址：**http://localhost:5500/**
- 如果端口冲突，Live Server 会自动使用其他端口

## 🌟 功能特性

### 1. 📋 简洁布局
- **左右分栏**：左侧文件树，右侧文档内容
- **文件导航**：直观的文件目录结构
- **无限高度**：文档内容区域无高度限制

### 2. 🔍 搜索功能
- **实时搜索**：输入关键词即时过滤文件列表
- **类型过滤**：按文件类型和名称搜索
- **搜索结果**：高亮显示匹配项目

### 3. 📖 文档阅读
- **Markdown 渲染**：完整支持 Markdown 语法
- **代码高亮**：支持代码块语法高亮
- **响应式阅读**：自动适应文档长度

### 4. ⚡ 便捷操作
- **双击选择**：点击左侧文件直接查看
- **键盘快捷键**：
  - `Ctrl/Cmd + K`：聚焦搜索框
  - `Ctrl/Cmd + F`：聚焦搜索框
  - `Enter`：搜索确认

## 📁 文档列表

站点自动扫描 `docs/` 目录下的所有 `.md` 文件，自动识别文档类型：

| 文档类型 | 识别关键词 | 图标 | 说明 |
|----------|------------|------|------|
| 需求文档 | prd、需求、product | 📋 | 产品需求与功能规划 |
| 架构设计 | architecture、arch、设计、executor、执行器 | 🏗️ | 系统架构与技术设计 |
| 开发指南 | guide、开发、development、plugin、插件 | 📖 | 开发配置与用户指南 |
| 流程规划 | flow、流程图、journey、用户体验、页面 | 🔄 | 业务流程与用户流程 |

**支持的文件列表**：
- 所有扩展名为 `.md` 的 Markdown 文件
- 自动从 `.filelist` 文件读取列表（可选）
- 支持动态文件扫描

## 🔧 自定义配置

### 添加新文档
1. 在 `docs/` 目录下添加新的 `.md` 文件
2. （可选）在 `.filelist` 文件中添加文件名（每行一个）
3. 刷新浏览器页面即可

推荐的文件命名模式（便于自动类型识别）：
- `产品需求文档.md` → 自动识别为需求文档
- `系统架构设计-v2.md` → 自动识别为架构设计
- `插件开发指南.md` → 自动识别为开发指南
- `页面流程图.md` → 自动识别为流程规划

### 修改主题配色
可以在 `index.html` 的 `<style>` 部分修改 CSS 变量：
```css
:root {
    --primary-color: #2563eb;    /* 主色调 */
    --secondary-color: #3b82f6;   /* 次要色 */
    --bg-color: #f8fafc;          /* 背景色 */
    --text-color: #1e293b;        /* 文本色 */
    --border-color: #e2e8f0;      /* 边框色 */
    --hover-color: #f1f5f9;       /* 悬停色 */
}
```

## 🔍 搜索功能

支持以下搜索方式：
1. **文件名搜索**：按文档文件名关键字
2. **类型搜索**：按文档类型（requirement/architecture/guide/plan）
3. **组合搜索**：支持多关键词同时搜索

**搜索提示**：
- 输入时实时显示搜索结果
- 清空搜索框可恢复完整列表
- 支持中文和英文搜索

## 📱 移动端支持

站点已完全适配移动设备：
- **响应式布局**：在小屏幕设备上自动切换为上下布局
- **触摸优化**：按钮和文件项适合触摸操作
- **滚动优化**：优化文档内容的滚动体验

### 移动端布局变化
1. **屏幕宽度 < 768px**：左右布局变为上下布局
2. **屏幕宽度 < 640px**：适配小屏幕移动设备
3. **触摸事件**：支持滑动和点击操作

## 🛠️ 开发与维护

### 文件结构
```
docs/
├── index.html                 # 主页面文件（左右布局）
├── docs.js                   # 文档扫描与显示逻辑
├── README.md                 # 本说明文件
├── 新布局使用说明.md         # 新版功能说明
├── .filelist                 # 可选的文档列表文件
├── *.md                      # 所有的 Markdown 文档
├── .vscode/settings.json     # Live Server 配置文件
└── *.code-workspace         # VS Code 工作区配置
```

### 动态文件扫描
系统会自动扫描 `docs/` 目录下的 `.md` 文件，无需手动维护文档列表。

### 修改样式
Markdown 渲染样式在 `index.html` 的 `.markdown-content` 类中定义。
布局样式在相应的 CSS 类中定义。

### 扩展功能
可以在 `docs.js` 中添加新功能，目前核心函数包括：
- `scanDocsDirectory()`：扫描文档目录
- `renderFileTree()`：渲染文件树
- `loadDocument()`：加载文档内容
- `searchDocuments()`：搜索功能

## 🚦 已知限制

1. **跨域限制**：浏览器安全策略限制，部分链接可能需要调整
2. **大文件性能**：超大 Markdown 文件可能会导致预览卡顿
3. **实时更新**：需要手动刷新页面才能看到新增文档
