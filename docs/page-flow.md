# 页面流程图 — API 编排平台

> 核心数据模型：**资源(Resource) → 工具(Tool) → 流程(Flow) → 触发器(Trigger)**  
> 所有流程图标注断点位置与优化方案。

---

## 1. 核心流程总览

```
┌─────────────────────────────────────────────────────────────────────────────┐
│                          用户核心操作路径                                     │
│                                                                             │
│  ① 创建资源 ──→ ② 创建/导入工具 ──→ ③ 发布工具 ──→ ④ 创建流程               │
│                                                    │                         │
│                                               ⑤ 编排节点                    │
│                                               ⑥ 配置映射                    │
│                                               ⑦ 调试测试                    │
│                                                    │                         │
│  ⑩ 监控运维 ←── ⑨ 触发器调用 ←── ⑧ 发布流程 + 创建触发器                     │
└─────────────────────────────────────────────────────────────────────────────┘
```

## 2. 流程一：创建资源

```mermaid
flowchart TD
    START([资源管理入口]) --> A[资源列表页 /resources]
    A --> B[点击 新建资源]
    B --> C[选择资源类型]
    C --> D{资源类型?}

    D -->|HTTP 服务| E[填写 HTTP 连接配置]
    D -->|gRPC 服务| F[填写 gRPC 连接配置]
    D -->|数据库| G[填写数据库连接配置]
    D -->|其他插件| H[插件特定配置表单]

    E --> I[测试连接]
    F --> I
    G --> I
    H --> I

    I --> J{连接测试}
    J -->|✅ 成功| K[创建资源]
    J -->|❌ 失败| L[展示错误信息]
    L --> I

    K --> M[跳转资源详情页]
    M --> N{下一步?}
    N -->|创建工具| O[流程二]
    N -->|返回列表| A

    style C fill:#6366F1,color:#fff
    style I fill:#22C55E,color:#fff
    style L fill:#EF4444,color:#fff
```

### 流程说明

| 步骤 | 页面 | 操作 | 反馈 |
|------|------|------|------|
| 选择类型 | 创建资源页 | 卡片式选择（HTTP/gRPC/DB/插件） | 选中态高亮 |
| 填写配置 | 创建资源页 | 根据类型展示不同表单 | 实时校验 |
| 测试连接 | 创建资源页 | 点击"测试连接"按钮 | 加载 → 成功✅ / 失败❌ + 错误详情 |
| 创建完成 | 资源详情页 | 自动跳转 | 成功 Toast |

## 3. 流程二：创建 & 导入工具

```mermaid
flowchart TD
    START([从资源详情/工具列表触发]) --> A{创建方式?}

    A -->|单个创建| B[工具编辑页 /tools/new]
    A -->|批量导入| C[导入向导 /tools/import]
    A -->|从资源详情进入| B

    %% 单个创建
    B --> D[选择所属资源]
    D --> E[填写基础信息]
    E --> F[配置输入参数]
    F --> G[配置输出参数]
    G --> H[发送测试]
    H --> I{测试结果?}
    I -->|通过| J[保存工具]
    I -->|失败| K[查看错误 + 修改]
    K --> F
    J --> L[工具列表更新]

    %% 批量导入
    C --> C1[选择导入方式]
    C1 --> C2{导入类型?}
    C2 -->|OpenAPI/Swagger| C3a[上传 yaml/json 文件]
    C2 -->|Proto 文件| C3b[上传 .proto 文件]
    C2 -->|SQLc 文件| C3c[上传 .sql 文件]

    C3a --> C4[解析 & 预览工具列表]
    C3b --> C4
    C3c --> C4

    C4 --> C5{预览确认?}
    C5 -->|确认导入| C6[批量创建工具]
    C5 -->|取消| A
    C6 --> C7[导入结果摘要]
    C7 --> L

    style C fill:#6366F1,color:#fff
    style C4 fill:#F59E0B,color:#fff
    style H fill:#22C55E,color:#fff
```

### 批量导入状态矩阵

| 导入类型 | 文件格式 | 解析产物 | 创建对象 |
|---------|---------|---------|---------|
| OpenAPI/Swagger | `.yaml` / `.json` | 解析 paths + operations | 每个操作 → 1 个工具 |
| Proto | `.proto` | 解析 services + methods | 每个 method → 1 个工具 |
| SQLc | `.sql` | 解析命名查询 | 每个查询 → 1 个工具 |

## 4. 流程三：流程编排（⭐ 核心流程）

```mermaid
flowchart TD
    START([流程管理入口]) --> A[流程列表 /flows]
    A --> B[点击 新建流程]
    B --> C[编排画布 /flows/new]

    C --> D[定义流程输入参数]
    D --> E{继续操作?}

    E -->|从左侧面板拖入工具节点| F[节点添加到画布]
    F --> G[配置节点参数映射]
    G --> H[流程输入/上游输出 → 工具输入参数]
    H --> I{操作?}

    I -->|拖入更多节点| F
    I -->|连接节点| J[创建连线 表达调用顺序]
    J --> K[配置节点间数据映射]
    K --> I
    I -->|定义返回值| L[指定流程输出]
    L --> I

    I -->|测试运行| M[底部测试面板滑出]
    M --> N[画布节点实时更新状态]
    N --> O{全部通过?}
    O -->|是| P[✅ 测试通过]
    O -->|否| Q[点击失败节点查看详情]
    Q --> R[查看请求/响应错误]
    R --> S{操作?}
    S -->|修改配置| G
    S -->|单节点重测| M
    S -->|全流程重测| M

    P --> T{操作?}
    T -->|保存| U[自动保存 + 版本更新]
    T -->|发布流程| V[流程四]

    style C fill:#6366F1,color:#fff
    style F fill:#22C55E,color:#fff
    style M fill:#4F46E5,color:#fff
    style Q fill:#EF4444,color:#fff
```

### 节点配置面板交互细节

```
选中画布节点 → 右侧滑出配置抽屉
├── 节点信息区
│   ├── 工具名称: 创建用户
│   ├── 所属资源: 用户服务 (HTTP)
│   └── 工具版本: v2
├── Tab: 参数映射
│   ├── 输入映射表
│   │   ├── 流程参数 username → 工具参数 name
│   │   ├── 流程参数 email → 工具参数 email
│   │   └── 上游节点.output.userId → 工具参数 referrerId
│   └── [+ 添加映射]
├── Tab: 高级设置
│   ├── 重试策略: 失败重试 3 次
│   ├── 超时时间: 30s
│   └── 条件执行: 仅当 upstream.success === true
└── 操作栏
    ├── [▶ 测试此节点]
    └── [保存配置]
```

## 5. 流程四：发布 & 版本管理

```mermaid
flowchart TD
    START([在编排画布/流程列表触发]) --> A{发布对象?}

    A -->|发布工具| B[工具发布流程]
    A -->|发布流程| C[流程发布流程]

    %% 工具发布
    B --> B1[工具详情页 点击发布]
    B1 --> B2[发布确认弹窗]
    B2 --> B3{是否首次发布?}
    B3 -->|是 首次发布| B4[工具状态 → 已发布 v1]
    B3 -->|否 更新发布| B5[自动检测下游依赖流程]
    B5 --> B6[展示影响分析]
    B6 --> B6a[依赖流程列表 + 工作空间归属]
    B6 --> B6b[接口变更对比 Breaking/Non-breaking]
    B6a --> B7[确认发布新版本]
    B7 --> B8[工具状态 → 已发布 v(n+1)]
    B4 --> B9[通知下游流程标记可升级]
    B8 --> B9
    B9 --> B10[流程编排画布中显示 🔄 升级提醒]
    B10 --> B11[开发者手动升级工具版本]
    B11 --> B12{参数兼容?}
    B12 -->|兼容 自动迁移| B13[节点升级完成 重新测试发布流程]
    B12 -->|不兼容| B14[展示参数差异 需手动调整映射]
    B14 --> B13
    B4 --> B5[其他工作空间的流程中可使用该工具]

    %% 流程发布
    C --> C1[编排画布 点击发布]
    C1 --> C2[发布确认弹窗]
    C2 --> C3[展示变更摘要]
    C3 --> C4[生成新版本号]
    C4 --> C5[流程状态 → 已发布]
    C5 --> C6[可被触发器绑定]

    %% 版本管理
    B4 --> D[版本历史]
    C5 --> D
    D --> D1[查看版本列表]
    D1 --> D2[对比两个版本]
    D2 --> D3{操作?}
    D3 -->|回滚| D4[确认回滚到指定版本]
    D3 -->|查看| D5[查看版本快照 只读]

    %% 下线保护
    B --> E[尝试下线工具]
    E --> E1{是否有流程引用该工具?}
    E1 -->|有引用| E2[阻止下线 展示依赖流程列表]
    E2 --> E3[需先在相关流程中移除该工具节点]
    E3 --> E4[再次尝试下线]
    E4 --> E1
    E1 -->|无引用| E5[确认下线]
    E5 --> E6[工具状态 → 已下线]

    style B2 fill:#F59E0B,color:#fff
    style C2 fill:#F59E0B,color:#fff
    style B4 fill:#22C55E,color:#fff
    style B5 fill:#3B82F6,color:#fff
    style B8 fill:#22C55E,color:#fff
    style C5 fill:#22C55E,color:#fff
    style E2 fill:#EF4444,color:#fff
```

> **关键约束**：
> - 只有已发布的工具才能在流程中被调用，只有已发布的流程才能被触发器绑定
> - 流程创建时锁定引用工具的具体版本，工具更新后需在编排画布中手动升级节点版本
> - 有流程引用的工具不允许下线，必须先在所有依赖流程中移除该工具节点

## 6. 流程五：创建触发器

```mermaid
flowchart TD
    START([触发器管理入口]) --> A[触发器列表 /triggers]
    A --> B[点击 新建触发器]
    B --> C[选择触发器类型]

    C --> D{触发器类型?}
    D -->|RESTful API| E[配置 API 路径和方法]
    D -->|定时器| F[配置 Cron 表达式]
    D -->|MQ 消费者| G[配置 Topic 和消费组]
    D -->|其他插件| H[插件特定配置]

    %% 绑定流程
    E --> I[绑定目标流程]
    F --> I
    G --> I
    H --> I

    I --> I1[仅展示已发布的流程列表]
    I1 --> J[选择流程]

    %% 输入映射
    J --> K[配置输入映射]
    K --> K1{触发器类型?}
    K1 -->|RESTful| K2[请求体/参数 → 流程输入参数]
    K1 -->|Timer| K3[定时触发参数 如执行时间戳]
    K1 -->|MQ| K4[消息体 → 流程输入参数]

    %% 输出映射
    K2 --> L[配置输出映射]
    K3 --> L
    K4 --> L

    L --> L1{触发器类型?}
    L1 -->|RESTful| L2[流程返回值 → HTTP 响应体]
    L1 -->|Timer| L3[无需输出配置]
    L1 -->|MQ| L4[流程返回值 → 回复消息/下一个Topic]

    L2 --> M[配置错误处理]
    L3 --> M
    L4 --> M

    M --> N{测试?}
    N -->|发送测试请求| O[运行流程并展示结果]
    O --> P{结果符合预期?}
    P -->|是| Q[创建触发器]
    P -->|否| R[调整映射配置]
    R --> K

    N -->|直接创建| Q
    Q --> S[触发器已创建 ✅]
    S --> T[触发器列表更新]

    style C fill:#6366F1,color:#fff
    style I1 fill:#F59E0B,color:#fff
    style K fill:#22C55E,color:#fff
    style L fill:#22C55E,color:#fff
    style O fill:#4F46E5,color:#fff
```

### 触发器映射配置说明

| 触发器类型 | 输入映射 | 输出映射 |
|-----------|---------|---------|
| **RESTful API** | HTTP 请求体/Header/Query → 流程输入参数 | 流程返回值 → HTTP 响应体/状态码 |
| **定时器** | 系统注入（触发时间戳、执行编号等）→ 流程输入 | 无（结果记录到运行日志） |
| **MQ 消费者** | 消息体/Header → 流程输入参数 | 流程返回值 → 回复消息或转发下一个 Topic |

## 8. 跨流程跳转矩阵

| 起始页面 | 目标页面 | 触发方式 | 携带上下文 |
|---------|---------|---------|-----------|
| 资源列表 → 资源详情 | 点击资源行 | 资源 ID |
